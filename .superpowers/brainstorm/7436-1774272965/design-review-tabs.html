What changes

  • New helper parseXlsxSheets(html) — splits the concatenated HTML string by <h3> tags into [{name, html}]. Pure function, no renderer changes.
  • Review.jsx — adds currentSheet state (sheet name string). When format is xlsx, parses html into sheets array, renders tab bar above viewer, sets innerHTML to active sheet only.
  • Tab switching — clicking a tab updates currentSheet; the effect that writes innerHTML re-runs. Chip overlay still works (applies to only the active sheet DOM).
  • After field insertion — insertXlsx re-renders to a new html string → re-parsed → currentSheet preserved by name.
  • Overflow — tab bar is overflow-x: auto for workbooks with many sheets.
  • DOCX unchanged — no tab bar rendered for docx files.

What stays the same

  • renderXlsx — no changes to renderer or its return shape
  • scanData — html field remains a plain string, no schema changes
  • Field insertion, save flow, popover — all unchanged
  • All existing tests — renderer tests unaffected; Review tests need new tab-click assertions
